Class EquipmentPotential

Namespace

Represents the latent ("potential") option layer attached to an Equipment. This is an independent layer that lives on top of the equipment's crafted stats and skills; it does not touch StatsMap so that each option keeps its own identity (row id + rolled value) and is fully decoupled from enhancement amplification.

Serialization is a self-contained List with its own version number, so the potential schema can evolve independently of the shared SerializationVersion. Field order (List format): [version, unlockedSlotCount, slots].

Remarks

Only the granting side is implemented in this phase: options are stored as (optionRowId, rolledValue) pairs, which are neutral with respect to how the option is later interpreted/applied to stats. Interpretation is intentionally out of scope here.
