Represents a sweep action that automatically completes multiple event dungeon battles without player interaction.

This action allows players to automatically repeat battles on already cleared event dungeon stages, consuming tickets and receiving rewards without manual combat interaction.

Key Features: - Only works on previously cleared stages (no stage clearing) - No ticket purchase functionality (uses existing tickets only) - Automatic reward and experience calculation - Multiple battle execution in a single action

This action modifies the following states: - AvatarState: Updates player's stats, inventory, and experience - EventDungeonInfo: Updates event dungeon progress and ticket usage - RuneState: Updates rune effects and bonuses - ItemSlotState: Updates equipped items - RuneSlotState: Updates equipped runes - CP: Updates character's combat power

Execution Flow: 1. Validate input parameters and avatar state 2. Load and validate game sheets and configurations 3. Validate event dungeon info and ticket availability 4. Check stage clearance requirements 5. Update rune and item slot states 6. Calculate combat power (CP) 7. Generate rewards and experience 8. Update avatar state and return modified world state

GetRewardItems(IRandom, int, Row, MaterialItemSheet)

Gets reward items for event dungeon sweep based on play count and stage configuration.

This method simulates multiple battle rewards without actually performing combat. It generates items based on the stage's drop configuration and the number of plays requested.

Process: 1. Determine the number of items to drop per play (random between min and max) 2. For each play, generate rewards using the stage's item selector 3. Collect all rewards and sort them by item ID for consistency
